# Setting Up Your RAG Engineering Environment

RAG (Retrieval-Augmented Generation) engineering sits at the intersection of information retrieval, embedding models, vector databases, and LLM APIs. Before building pipelines, you need every piece of that stack installed, tested, and wired together. This lesson does exactly that.

## Install Python with pyenv

```bash
# macOS
brew install pyenv
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.zshrc
echo '[[ -d $PYENV_ROOT/bin ]] && export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.zshrc
echo 'eval "$(pyenv init -)"' >> ~/.zshrc
source ~/.zshrc
pyenv install 3.12.3 && pyenv global 3.12.3

# Linux
curl https://pyenv.run | bash
# Add PYENV_ROOT exports to ~/.bashrc, then:
pyenv install 3.12.3 && pyenv global 3.12.3

# Verify
python --version   # Python 3.12.3
```

## Install uv and Create the Project

```bash
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

uv init rag-engineering-course
cd rag-engineering-course
```

Install all RAG stack dependencies:

```bash
uv add groq \
        sentence-transformers \
        chromadb \
        langchain-text-splitters \
        ragas \
        datasets \
        python-dotenv \
        jupyterlab ipykernel
```

> **Note:** `sentence-transformers` downloads model weights on first use from Hugging Face Hub. The first embedding call will take 10-60 seconds depending on your connection; subsequent calls are instant because the model is cached at `~/.cache/huggingface/`.

## OpenTelemetry for Tracing

Production RAG systems need tracing to understand which retrieval steps are slow, which prompts are failing, and how latency breaks down across components. OpenTelemetry is the industry-standard observability framework.

```bash
uv add opentelemetry-sdk opentelemetry-exporter-otlp-proto-grpc
```

Set up a basic tracer:

```python
# src/tracing.py
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor, ConsoleSpanExporter

def setup_tracing(service_name: str = "rag-pipeline") -> trace.Tracer:
    """Configure OpenTelemetry tracing with console exporter for development."""
    provider = TracerProvider()
    # In development: export spans to console
    # In production: swap ConsoleSpanExporter for OTLPSpanExporter
    processor = BatchSpanProcessor(ConsoleSpanExporter())
    provider.add_span_processor(processor)
    trace.set_tracer_provider(provider)
    return trace.get_tracer(service_name)

tracer = setup_tracing()
```

Use it to instrument your RAG functions:

```python
with tracer.start_as_current_span("retrieval") as span:
    span.set_attribute("query", user_query)
    span.set_attribute("top_k", 5)
    results = retrieve(user_query, collection, embed_model, top_k=5)
    span.set_attribute("chunks_returned", len(results))
```

## End-to-End Stack Verification

Run this script to verify every component of the RAG stack:

```python
# test_rag_setup.py
import os
import sys
from dotenv import load_dotenv

load_dotenv()

print("RAG Engineering Environment Verification")
print("=" * 50)

# 1. Check API key
api_key = os.getenv("GROQ_API_KEY")
if not api_key:
    print("FAIL: GROQ_API_KEY not found in .env")
    sys.exit(1)
print("PASS: GROQ_API_KEY found")

# 2. Test sentence-transformers embedding
from sentence_transformers import SentenceTransformer
import numpy as np

print("\nLoading embedding model (first run downloads weights)...")
model = SentenceTransformer("BAAI/bge-small-en-v1.5")
embedding = model.encode("Hello, RAG world!", normalize_embeddings=True)
print(f"PASS: Embedding shape: {embedding.shape}, dtype: {embedding.dtype}")

# 3. Test ChromaDB
import chromadb

client = chromadb.Client()
collection = client.create_collection("test_collection")

docs = [
    "RAG combines retrieval with generation.",
    "Vector databases store embeddings for fast search.",
    "Groq provides fast LLM inference.",
]
embeddings = model.encode(docs, normalize_embeddings=True).tolist()
collection.add(
    documents=docs,
    embeddings=embeddings,
    ids=["doc_0", "doc_1", "doc_2"],
)

query_embedding = model.encode(
    "Represent this sentence for searching relevant passages: what is RAG?",
    normalize_embeddings=True,
).tolist()

results = collection.query(query_embeddings=[query_embedding], n_results=2)
print(f"PASS: ChromaDB retrieved {len(results['documents'][0])} chunks")
print(f"      Top result: '{results['documents'][0][0][:60]}...'")

# 4. Test Groq API
from groq import Groq

groq_client = Groq()
response = groq_client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=[{"role": "user", "content": "Reply with exactly: GROQ_OK"}],
    max_tokens=10,
    temperature=0,
)
reply = response.choices[0].message.content.strip()
print(f"PASS: Groq API response: '{reply}'")

# 5. Report
print("\n" + "=" * 50)
print("All components verified successfully.")
print("You are ready to build RAG pipelines.")
```

## Summary

- Install Python via **pyenv** and manage dependencies with **uv**; every package in the RAG stack (groq, sentence-transformers, chromadb, langchain-text-splitters, ragas) installs cleanly with a single `uv add` command.
- Store your **Groq API key** in `.env` loaded with `python-dotenv`; add `.env` to `.gitignore` and commit `.env.example` as a template for teammates.
- The `test_rag_setup.py` verification script tests all four components end-to-end: embedding a sentence, storing it in ChromaDB, querying by similarity, and calling the Groq API — run it before moving to Lesson 1.
- Add **OpenTelemetry** tracing from the start; instrument retrieval and generation spans so you can measure latency and debug slow queries in development before they become production incidents.
- Exclude `chroma_db/` from git (it is a binary directory); re-index from source documents as part of your pipeline rather than committing the vector store.
