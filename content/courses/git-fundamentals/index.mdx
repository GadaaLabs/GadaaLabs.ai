Git is the version control system that powers virtually every professional software project on the planet. It is the tool that lets teams of ten — or ten thousand — collaborate on the same codebase without stepping on each other. It is the safety net that lets you experiment fearlessly, the time machine that lets you recover any past state of your project, and the communication layer that turns a sequence of changes into a readable engineering history.

This course teaches Git from first principles. It does not assume you have used any version control system before. It also does not stop at the basics: by the end of the course you will understand Git's internal object model, know when to rebase vs merge, be able to resolve complex conflicts, and understand the workflow patterns used by high-performing engineering teams at scale.

Each lesson is built around practical exercises using a real project — a command-line task manager called `taskr` — that grows in complexity across the ten lessons. Every concept is illustrated with the actual commands you will type and the actual output you will see. There are no toy examples and no hand-waving. You will master Git the way professionals use it: by understanding what it does under the hood, not just memorising command sequences.

The course is structured in three natural arcs. The first arc (lessons 1–3) covers the foundation: what Git is, how to create and commit to a repository, and how branching works. The second arc (lessons 4–7) covers the power tools: undoing changes, working with remote repositories, resolving conflicts, and rewriting history with rebase and cherry-pick. The third arc (lessons 8–10) covers the professional layer: tagging releases, understanding Git's internals for deep debugging, and choosing the right workflow pattern for your team.

Take the lessons in order. Each one builds on the previous. By the end, Git will not feel like a collection of arcane commands — it will feel like a coherent, elegant system you understand from the inside out.
