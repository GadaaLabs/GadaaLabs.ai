## How Python Stores Collections

Every time you store a group of values, you make a performance and semantic decision. Python's four built-in containers — **list**, **tuple**, **dict**, and **set** — each have distinct internal implementations, time complexities, and intended purposes.

Choosing the wrong container does not just slow your code — it obscures your intent and leads to bugs. This lesson gives you a complete mental model of each.

---

## Lists — Dynamic Arrays of References

A list is a **mutable, ordered, dynamically resized array of object references**. CPython stores an array of pointers to `PyObject` structs. This means:

- **Index access** → O(1) (direct pointer arithmetic)
- **Append** → O(1) amortised (over-allocates to avoid frequent resizing)
- **Insert at front** → O(n) (all pointers shift)
- **Search (`in`)** → O(n) (linear scan)

### List as Stack and Queue — When to Use Each

<PythonPlayground code={String.raw`# STACK — Last In, First Out (LIFO)
# list.append() to push, list.pop() to pull — BOTH O(1)

call_stack = []
def call(fn_name):
    call_stack.append(fn_name)
    print(f"  CALL  {fn_name}  | stack: {call_stack}")

def ret():
    fn = call_stack.pop()
    print(f"  RETURN {fn} | stack: {call_stack}")

call("main")
call("read_file")
call("parse_line")
ret()
ret()
ret()

# QUEUE — First In, First Out (FIFO)
# PROBLEM: list.insert(0, x) is O(n) — every element shifts!
# SOLUTION: collections.deque  — O(1) at BOTH ends

from collections import deque

task_queue = deque()

def enqueue(task):
    task_queue.append(task)    # add to right — O(1)
    print(f"  Enqueued: {task!r}  queue: {list(task_queue)}")

def process():
    if task_queue:
        task = task_queue.popleft()   # remove from left — O(1)
        print(f"  Processing: {task!r}  remaining: {list(task_queue)}")

print("\nTask Queue:")
enqueue("send_email")
enqueue("resize_image")
enqueue("generate_report")
process()
process()

# deque with maxlen — sliding window, auto-evicts oldest
recent = deque(maxlen=4)
print("\nSliding window (maxlen=4):")
for val in [10, 20, 30, 40, 50, 60]:
    recent.append(val)
    print(f"  Added {val}: {list(recent)}")

A tuple is an **immutable, ordered sequence**. Because it cannot change, Python stores it more compactly than a list and it can be used as a **dictionary key** or **set element**.

## Dictionaries — Hash Tables with Ordering

A dict is an **ordered** (Python 3.7+), **mutable hash table**. Keys are hashed to find a bucket — O(1) average for get/set/delete.

**Key requirements**: must be **hashable** — immutable types work (int, str, float, tuple of hashables, frozenset). Lists and dicts cannot be keys.

## When to Use Each Data Structure

| Need | Best choice | Why |
|------|-------------|-----|
| Ordered sequence, random access | `list` | O(1) index |
| Queue (FIFO) | `deque` | O(1) popleft |
| Stack (LIFO) | `list` | O(1) pop |
| Key-value store | `dict` | O(1) lookup |
| Membership testing (repeated) | `set` | O(1) `in` |
| Immutable record / dict key | `tuple` | hashable |
| Frequency counting | `Counter` | built-in arithmetic |
| Grouping without "if key in" | `defaultdict` | auto-initialise |

<PythonPlayground code={String.raw`from collections import defaultdict, Counter

class ContactBook:
    def __init__(self):
        self._contacts  = {}             # id -> contact dict
        self._next_id   = 1
        self._tag_index = defaultdict(set)  # tag -> set of ids

    def add(self, name, email="", phone="", tags=None):
        tags = set(tags or [])
        contact = {"id": self._next_id, "name": name,
                   "email": email, "phone": phone, "tags": tags}
        self._contacts[self._next_id] = contact
        for tag in tags:
            self._tag_index[tag].add(self._next_id)
        self._next_id += 1
        return contact["id"]

    def get(self, contact_id):
        return self._contacts.get(contact_id)

    def update(self, contact_id, **kwargs):

<PythonPlayground code={String.raw`# Implement an LRU (Least Recently Used) Cache with O(1) get and put.
# When at capacity, evict the least recently used item.
# Use OrderedDict: it tracks insertion order and supports move_to_end()

from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.cap   = capacity
        self.cache = OrderedDict()  # key -> value, ordered by recency

    def get(self, key):
        """Return value if found; mark it as recently used. Return -1 if absent."""
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)   # mark as most recently used
        return self.cache[key]

    def put(self, key, value):
        """Insert key-value. Evict LRU item if over capacity."""
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.cap:
            self.cache.popitem(last=False)  # remove OLDEST (front)

### Exercise 4 — BFS Shortest Path (Graph as dict of sets)

<PythonPlayground code={String.raw`from collections import defaultdict, deque

class Graph:
    def __init__(self):
        self.adj = defaultdict(set)

    def add_edge(self, u, v, directed=False):
        self.adj[u].add(v)
        if not directed:
            self.adj[v].add(u)

    def bfs_shortest_path(self, start, end):
        """Return list of nodes forming the shortest path from start to end."""
        if start == end:
            return [start]
        visited = {start}
        queue = deque([[start]])    # queue of PATHS
        while queue:
            path = queue.popleft()
            node = path[-1]
            for neighbor in sorted(self.adj[node]):  # sorted for determinism
                if neighbor == end:
                    return path + [neighbor]
                if neighbor not in visited:
                    visited.add(neighbor)
                    queue.append(path + [neighbor])
        return []   # no path found

    def connected_components(self):
        """Return list of sets, each set is one connected component."""
        visited   = set()
        components = []
        for node in self.adj:
            if node not in visited:
                component = set()
                q = deque([node])
                while q:
                    n = q.popleft()
                    if n not in visited:
                        visited.add(n)
                        component.add(n)
                        q.extend(self.adj[n] - visited)
                components.append(component)
        return components

<PythonPlayground code={String.raw`from collections import deque

def sliding_window_max(nums, k):
    """
    Return the maximum value in every window of size k.
    Naive: O(n*k). Optimal with deque: O(n).

    Key insight: maintain a deque of INDICES where values are
    in DECREASING order. The front of the deque is always the
    max of the current window.
    """
    result = []
    dq = deque()   # stores indices; front = index of current max

    for i, num in enumerate(nums):
        # Remove indices outside this window
        while dq and dq[0] < i - k + 1:
            dq.popleft()
        # Remove indices of smaller values — they can never be the max
        while dq and nums[dq[-1]] < num:
            dq.pop()
        dq.append(i)
        # Window is full once i >= k-1
        if i >= k - 1:
            result.append(nums[dq[0]])

    return result

print(sliding_window_max([1,3,-1,-3,5,3,6,7], k=3))  # [3,3,5,5,6,7]
print(sliding_window_max([1,3,1,2,0,5],       k=3))  # [3,3,2,5]
print(sliding_window_max([4,3,2,1],            k=2))  # [4,3,2]

<PythonPlayground code={String.raw`# A sparse matrix stores only non-zero values.
# Representing it as dict-of-dicts uses far less memory than a full 2D list.

class SparseMatrix:
    def __init__(self, rows, cols):
        self.rows = rows
        self.cols = cols
        self._data = {}   # {row: {col: value}}

    def set(self, r, c, val):
        if val == 0:
            # Remove zero entries to keep it truly sparse
            if r in self._data and c in self._data[r]:
                del self._data[r][c]
            return
        if r not in self._data:
            self._data[r] = {}
        self._data[r][c] = val

    def get(self, r, c):
        return self._data.get(r, {}).get(c, 0)

    def add(self, other):
        result = SparseMatrix(self.rows, self.cols)
        all_rows = set(self._data) | set(other._data)
        for r in all_rows:
            all_cols = set(self._data.get(r, {})) | set(other._data.get(r, {}))
            for c in all_cols:
                result.set(r, c, self.get(r, c) + other.get(r, c))
        return result

    def transpose(self):
        result = SparseMatrix(self.cols, self.rows)
        for r, cols in self._data.items():
            for c, val in cols.items():
                result.set(c, r, val)
        return result

    def nnz(self):
        return sum(len(cols) for cols in self._data.values())

    def display(self):
        for r in range(self.rows):
            row_str = " ".join(f"{self.get(r, c):3}" for c in range(self.cols))
            print(f"  [{row_str}]")

## Key Takeaways

- **Lists** are O(1) at the tail and by index — use `deque` for O(1) operations at both ends
- **Tuples** signal immutability and are hashable — use them as dict keys and to represent fixed records
- **Dict** lookups, inserts, and deletes are O(1) average — always prefer a dict over a list for key-based access
- **Sets** give O(1) membership testing — use them whenever you repeatedly ask "is X in collection?"
- **`defaultdict`** eliminates "if key not in dict" boilerplate for grouping, counting, and graph adjacency
- **`Counter`** supports frequency counting, arithmetic on counts, and `most_common(n)`
- **Comprehensions** are faster than for-loops for building collections; they express intent clearly
- **Generator expressions** produce values lazily — prefer them when you need to iterate only once
- The **`key=`** parameter in `sorted()` makes multi-key sorting trivial with no auxiliary data structures
- Choose your container by asking: "What operation will I do most often?" — then pick the O(1) one
