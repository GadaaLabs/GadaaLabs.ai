## Functions Are First-Class Objects

In Python, functions are **first-class objects** — they are values just like integers or strings. You can assign a function to a variable, store it in a list, pass it as an argument, and return it from another function. This is not a trick or an edge case; it is a core design principle of Python that unlocks enormous expressive power.

<PythonPlayground code={String.raw`def greet(name):
    """Return a personalized greeting."""
    return f"Hello, {name}!"

print(f"Identity: {id(greet)}")
print(f"Name:     {greet.__name__}")
print(f"Docstring:{greet.__doc__}")

# Assign to another name -- both point to the same function object
say_hello = greet
print(f"\nsay_hello is greet: {say_hello is greet}")
print(say_hello("Alice"))

### `def`, `return`, and the Implicit `None`

Every Python function returns a value. If you do not write a `return` statement (or write `return` with no value), Python implicitly returns `None`. This is a common source of bugs — assigning the result of a function that forgets to return.

### Default Values — The Mutable Default Trap

Default values are evaluated **once** when the `def` statement executes — NOT each time the function is called. Mutable defaults (lists, dicts) are shared across all calls. This is one of Python's most famous gotchas.

<PythonPlayground code={String.raw`# THE TRAP: mutable default argument
def add_item_BAD(item, collection=[]):
    collection.append(item)
    return collection

print(add_item_BAD("apple"))   # ['apple']
print(add_item_BAD("banana"))  # ['apple', 'banana'] -- NOT ['banana']!
print(add_item_BAD("cherry"))  # ['apple', 'banana', 'cherry']

# The default list is shared across ALL calls -- it persists between calls!
print(f"\nThe default list: {add_item_BAD.__defaults__}")

# THE FIX: use None as sentinel, create a new list inside the function
def add_item_GOOD(item, collection=None):
    if collection is None:
        collection = []         # new list created on each call
    collection.append(item)
    return collection

print("\nFixed version:")
print(add_item_GOOD("apple"))   # ['apple']
print(add_item_GOOD("banana"))  # ['banana']  -- correct!
print(add_item_GOOD("cherry"))  # ['cherry']  -- correct!

# Passing an explicit list still works
my_list = [1, 2]
print(add_item_GOOD(3, my_list))   # [1, 2, 3]

## Scope and the LEGB Rule

When Python encounters a name like `x`, it searches for it in a specific order: **L**ocal → **E**nclosing → **G**lobal → **B**uilt-in. The first match wins.

<PythonPlayground code={String.raw`x = "global"          # Global scope

def outer():
    x = "enclosing"   # Enclosing scope (for inner)

    def inner():
        x = "local"   # Local scope
        print(f"inner sees: {x}")   # "local"

    inner()
    print(f"outer sees: {x}")       # "enclosing"

outer()
print(f"module sees: {x}")          # "global"

# Built-in scope -- Python's built-in names live here
print(f"\nBuilt-in 'len': {len}")

# You can (but should not) shadow built-ins
def bad_example():
    len = lambda x: "I broke len!"   # shadows the built-in
    print(len([1, 2, 3]))             # calls our lambda, not the built-in

bad_example()
print(len([1, 2, 3]))                 # module scope still has the real len

# The LEGB lookup at work:
y = "global y"

def show_legb():
    # y is not defined locally or in enclosing scope
    # Python looks up LEGB and finds it in Global
    print(f"Found y in global scope: {y}")

show_legb()

# Built-in as fallback
def show_builtin():
    # 'print' is not defined locally, globally, or in any enclosing scope
    # Python finds it in the Built-in scope
    print("Found 'print' in built-in scope")

show_builtin()

### `global` and `nonlocal` Statements

Without a declaration, assignment inside a function creates a **local** variable, even if a global with the same name exists. Use `global` or `nonlocal` to modify outer variables.

    counter += 1

## Closures — The Deep Dive

A **closure** is a function that remembers the variables from the scope in which it was created, even after that scope has finished executing. It "closes over" those free variables, keeping them alive in special **cell objects**.

<PythonPlayground code={String.raw`def make_multiplier(n):
    # n lives in the enclosing scope of inner
    def inner(x):
        return x * n    # inner captures n from the enclosing scope
    return inner        # return inner WITH its reference to n

double = make_multiplier(2)
triple = make_multiplier(3)
times10 = make_multiplier(10)

print(double(5))    # 10
print(triple(5))    # 15
print(times10(7))   # 70

# Each closure has its own captured 'n' stored in a cell object
print(f"\ndouble.__closure__:  {double.__closure__}")
print(f"double's n:          {double.__closure__[0].cell_contents}")
print(f"triple's n:          {triple.__closure__[0].cell_contents}")

# The free variables a closure references
print(f"\ndouble.__code__.co_freevars: {double.__code__.co_freevars}")

# Practical closure: private state (pre-OOP style)
def make_bank_account(balance=0):
    _balance = balance

    def deposit(amount):
        nonlocal _balance
        if amount <= 0:
            raise ValueError("Deposit must be positive")
        _balance += amount
        return _balance

    def withdraw(amount):
        nonlocal _balance
        if amount > _balance:
            raise ValueError(f"Insufficient funds: have {_balance}, want {amount}")
        _balance -= amount
        return _balance

    def get_balance():
        return _balance

    return deposit, withdraw, get_balance

deposit, withdraw, balance = make_bank_account(100)
print(f"\nBalance: {balance()}")
print(f"After deposit 50:   {deposit(50)}")
print(f"After withdraw 30:  {withdraw(30)}")
print(f"Final balance:      {balance()}")

### The Loop Variable Capture Gotcha and Its Fix

Closures capture **variables**, not values. In a loop, all closures share the same loop variable — after the loop ends, they all see its final value.

<PythonPlayground code={String.raw`# THE TRAP: loop variable capture -- late binding
funcs_BAD = []
for i in range(5):
    funcs_BAD.append(lambda x: x + i)   # all capture the same 'i' variable

# After the loop, i == 4, so ALL lambdas add 4
results = [fn(10) for fn in funcs_BAD]
print(f"Bad (late binding): {results}")   # [14, 14, 14, 14, 14]

# FIX 1: default argument -- captures the value at definition time
funcs_DEFAULT = []
for i in range(5):
    funcs_DEFAULT.append(lambda x, captured=i: x + captured)

results = [fn(10) for fn in funcs_DEFAULT]
print(f"Fixed (default arg): {results}")  # [10, 11, 12, 13, 14]

# FIX 2: factory function -- most explicit and readable
def make_adder(n):
    def adder(x):
        return x + n
    return adder

funcs_FACTORY = [make_adder(i) for i in range(5)]
results = [fn(10) for fn in funcs_FACTORY]
print(f"Fixed (factory): {results}")      # [10, 11, 12, 13, 14]

# FIX 3: immediately-invoked lambda (unusual but valid)
funcs_IIFE = [(lambda n: lambda x: x + n)(i) for i in range(5)]
results = [fn(10) for fn in funcs_IIFE]
print(f"Fixed (IIFE):    {results}")      # [10, 11, 12, 13, 14]

## Lambda Functions

Lambdas are **anonymous single-expression functions**. They are syntactically restricted: one expression, no statements, no assignments. Use them when a full `def` would be overkill — primarily as the `key=` argument to sort functions or with `map`/`filter`.

## Recursion — When and How

A recursive function calls itself. Every recursive solution has two parts: a **base case** that stops the recursion, and a **recursive case** that reduces the problem to a smaller version of itself.

<PythonPlayground code={String.raw`import sys

# Classic: factorial
def factorial_recursive(n):
    if n <= 1:          # base case
        return 1
    return n * factorial_recursive(n - 1)   # recursive case

def factorial_iterative(n):
    result = 1
    for i in range(2, n + 1):
        result *= i
    return result

print(f"10! recursive: {factorial_recursive(10)}")
print(f"10! iterative: {factorial_iterative(10)}")

# Visualize the call stack
def factorial_traced(n, depth=0):
    indent = "  " * depth
    print(f"{indent}factorial({n}) called")
    if n <= 1:
        print(f"{indent}base case -> 1")
        return 1
    result = n * factorial_traced(n - 1, depth + 1)
    print(f"{indent}factorial({n}) = {n} * ... = {result}")
    return result

print("\nTraced factorial(4):")
factorial_traced(4)

# Python's recursion limit
print(f"\nRecursion limit: {sys.getrecursionlimit()}")

# Binary search -- a natural recursive algorithm
def binary_search(arr, target, low=0, high=None):
    if high is None:
        high = len(arr) - 1
    if low > high:
        return -1    # not found
    mid = (low + high) // 2
    if arr[mid] == target:
        return mid
    elif arr[mid] < target:
        return binary_search(arr, target, mid + 1, high)
    else:
        return binary_search(arr, target, low, mid - 1)

unique = sorted(set(data))
print(f"\nData: {unique}")
for target in [5, 7, 1]:
    idx = binary_search(unique, target)
    print(f"binary_search({target}) = {idx}")

### Memoization with `lru_cache`

Without memoization, recursive Fibonacci has exponential time complexity because it recomputes the same subproblems millions of times. With memoization, each subproblem is solved exactly once.

<PythonPlayground code={String.raw`from functools import lru_cache
import time

# WITHOUT memoization: recomputes fib(n-2) for every single call
call_count = [0]

def fib_slow(n):
    call_count[0] += 1
    if n < 2: return n
    return fib_slow(n-1) + fib_slow(n-2)

call_count[0] = 0
result = fib_slow(30)
print(f"fib_slow(30) = {result}")
print(f"Function calls: {call_count[0]:,}")

# WITH lru_cache: each value computed exactly once
@lru_cache(maxsize=None)
def fib_fast(n):
    if n < 2: return n
    return fib_fast(n-1) + fib_fast(n-2)

result = fib_fast(100)
info = fib_fast.cache_info()
print(f"\nfib_fast(100) = {result}")
print(f"Cache: hits={info.hits}, misses={info.misses}")
print(f"Total calls needed: {info.misses} (one per unique n)")

# Timing comparison for fib(30)
t0 = time.perf_counter()
fib_slow(30)
slow_ms = (time.perf_counter() - t0) * 1000

fib_fast.cache_clear()

fib_fast(30)
fast_ms = (time.perf_counter() - t0) * 1000

print(f"fib(30) with cache:    {fast_ms:.4f}ms")
if fast_ms > 0:

    if n < 2: return n
    a, b = 0, 1
    for _ in range(n - 1):
        a, b = b, a + b
    return b

### Tower of Hanoi — Classic Recursive Problem

<PythonPlayground code={String.raw`def hanoi(n, source, target, auxiliary, moves=None):
    """
    Move n disks from source peg to target peg using auxiliary peg.
    Key insight: to move n disks,
    1. Move top n-1 disks to auxiliary (out of the way)

## Project: Recursive Expression Calculator

This project implements a complete arithmetic expression calculator using **recursive descent parsing** — the same technique used in real compilers and interpreters. It handles operator precedence, parentheses, and unary negation.

The grammar:
```
expression := term (('+' | '-') term)*
term       := factor (('*' | '/') factor)*
factor     := unary ('^' unary)*
unary      := '-' unary | primary
primary    := number | '(' expression ')'
```

Each grammar rule maps directly to one recursive function — that is what makes this technique so elegant.

<PythonPlayground code={String.raw`class Token:
    def __init__(self, kind, value):
        self.kind  = kind
        self.value = value
    def __repr__(self):
        return f"Token({self.kind}, {self.value!r})"

def tokenize(text):
    tokens = []
    i = 0
    while i < len(text):
        ch = text[i]
        if ch.isspace():
            i += 1
        elif ch.isdigit() or (ch == "." and i+1 < len(text) and text[i+1].isdigit()):
            j = i
            while j < len(text) and (text[j].isdigit() or text[j] == "."):
                j += 1
            tokens.append(Token("NUM", float(text[i:j])))
            i = j
        elif ch == "+": tokens.append(Token("PLUS",   ch)); i += 1
        elif ch == "-": tokens.append(Token("MINUS",  ch)); i += 1
        elif ch == "*": tokens.append(Token("MUL",    ch)); i += 1
        elif ch == "/": tokens.append(Token("DIV",    ch)); i += 1
        elif ch == "^": tokens.append(Token("POW",    ch)); i += 1
        elif ch == "(": tokens.append(Token("LPAREN", ch)); i += 1
        elif ch == ")": tokens.append(Token("RPAREN", ch)); i += 1
        else: raise ValueError(f"Unknown character: {ch!r}")
    tokens.append(Token("EOF", None))
    return tokens

class Parser:
    def __init__(self, tokens):
        self.tokens = tokens
        self.pos    = 0

    def peek(self):
        return self.tokens[self.pos]

    def eat(self, kind):
        tok = self.peek()
        if tok.kind != kind:
            raise ValueError(f"Expected {kind}, got {tok.kind!r} ({tok.value!r})")
        self.pos += 1
        return tok

    def parse(self):
        result = self.expression()
        self.eat("EOF")
        return result

    # Grammar rule: expression := term (('+' | '-') term)*
    def expression(self):
        result = self.term()
        while self.peek().kind in ("PLUS", "MINUS"):
            if self.peek().kind == "PLUS":
                self.eat("PLUS");  result += self.term()
            else:
                self.eat("MINUS"); result -= self.term()
        return result

    # Grammar rule: term := factor (('*' | '/') factor)*
    def term(self):
        result = self.factor()
        while self.peek().kind in ("MUL", "DIV"):
            if self.peek().kind == "MUL":
                self.eat("MUL"); result *= self.factor()
            else:
                self.eat("DIV")
                right = self.factor()
                if right == 0: raise ZeroDivisionError("Division by zero")
                result /= right
        return result

    # Grammar rule: factor := unary ('^' unary)*  [right-associative]
    def factor(self):
        base = self.unary()
        if self.peek().kind == "POW":
            self.eat("POW")
            exp = self.factor()   # right-recursive gives right-associativity
            return base ** exp
        return base

    # Grammar rule: unary := '-' unary | primary
    def unary(self):
        if self.peek().kind == "MINUS":
            self.eat("MINUS")
            return -self.unary()  # recursive: handles --5 = 5
        return self.primary()

    # Grammar rule: primary := number | '(' expression ')'
    def primary(self):
        tok = self.peek()
        if tok.kind == "NUM":
            self.eat("NUM")
            return tok.value
        elif tok.kind == "LPAREN":
            self.eat("LPAREN")
            result = self.expression()
            self.eat("RPAREN")
            return result
        raise ValueError(f"Unexpected token: {tok!r}")

def calc(expr):
    return Parser(tokenize(expr)).parse()

    ("-5 + 3",                 -2),
    ("--5",                    5),
    ("100 / 4 / 5",            5),
    ("3.14 * 2 * 2",           12.56),
    ("2 * (3 + (4 - 1))",      12),
    ("(2+3)*(4-1)^2",          45),
