# Setting Up Your AI Automation Environment

AI automation systems — agents, multi-step workflows, automated pipelines — have stricter reliability requirements than one-off scripts. They run unattended, handle failures gracefully, and produce structured, observable outputs. Your environment needs to be set up with observability and reliability tooling from day one. This lesson builds that foundation.

## Install Python with pyenv

```bash
# macOS
brew install pyenv
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.zshrc
echo '[[ -d $PYENV_ROOT/bin ]] && export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.zshrc
echo 'eval "$(pyenv init -)"' >> ~/.zshrc
source ~/.zshrc
pyenv install 3.12.3 && pyenv global 3.12.3

# Linux
curl https://pyenv.run | bash
# Add exports to ~/.bashrc, then:
pyenv install 3.12.3 && pyenv global 3.12.3

# Verify
python --version   # Python 3.12.3
```

## Structlog: Structured JSON Logging

Agent systems make many sequential decisions. Without structured logs, debugging a 10-step agent pipeline is nearly impossible. Structlog outputs machine-readable JSON with every decision, tool call, and error.

```python
# src/logging_config.py
import logging
import structlog

def configure_logging(level: str = "INFO") -> None:
    shared_processors = [
        structlog.stdlib.add_log_level,
        structlog.stdlib.add_logger_name,
        structlog.processors.TimeStamper(fmt="iso"),
        structlog.processors.StackInfoRenderer(),
        structlog.processors.ExceptionRenderer(),
    ]

    if level == "DEBUG":
        renderer = structlog.dev.ConsoleRenderer()
    else:
        renderer = structlog.processors.JSONRenderer()

    structlog.configure(
        processors=shared_processors + [renderer],
        wrapper_class=structlog.make_filtering_bound_logger(
            getattr(logging, level.upper())
        ),
        context_class=dict,
        logger_factory=structlog.PrintLoggerFactory(),
    )

# Usage
from src.logging_config import configure_logging
configure_logging(level="INFO")
logger = structlog.get_logger()

logger.info("agent_step", step="tool_call", tool="web_search", query="AI news 2026")
logger.warning("retry", attempt=2, error="rate_limit", wait_seconds=4)
```

## OpenTelemetry: Distributed Tracing

Tracing gives you a timeline of every operation inside your agent — which tool calls happened, how long each took, and where errors occurred.

```python
# src/tracing.py
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor, ConsoleSpanExporter

def setup_tracing(service_name: str = "ai-automation") -> trace.Tracer:
    provider = TracerProvider()
    processor = BatchSpanProcessor(ConsoleSpanExporter())
    provider.add_span_processor(processor)
    trace.set_tracer_provider(provider)
    return trace.get_tracer(service_name)

tracer = setup_tracing()

# Example: trace an agent step
def call_tool(tool_name: str, inputs: dict) -> dict:
    with tracer.start_as_current_span(f"tool.{tool_name}") as span:
        span.set_attribute("tool.name", tool_name)
        span.set_attribute("tool.inputs", str(inputs))
        result = {"status": "ok", "output": "..."}
        span.set_attribute("tool.status", result["status"])
        return result
```

## n8n Workflow Automation: Docker Setup

n8n is an open-source visual workflow automation platform. It runs HTTP webhooks, schedules, and triggers that connect to your Python agents and external APIs — without writing integration glue code.

Create `docker-compose.yml` in your project root:

```yaml
version: "3.9"

services:
  n8n:
    image: docker.n8n.io/n8nio/n8n:latest
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=changeme
      - N8N_HOST=localhost
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
      - WEBHOOK_URL=http://localhost:5678/
      - GENERIC_TIMEZONE=UTC
    volumes:
      - n8n_data:/home/node/.n8n

volumes:
  n8n_data:
```

Start n8n:

```bash
docker compose up -d
# n8n UI at http://localhost:5678
# Default login: admin / changeme
```

> **Note:** Change the `N8N_BASIC_AUTH_PASSWORD` before exposing n8n on any network. For production, add TLS termination via a reverse proxy (Caddy or nginx) in front of the n8n container.

## Verify the Full Stack

```python
# test_automation_setup.py
import os
import sys
import structlog
from dotenv import load_dotenv

load_dotenv()

# Configure logging first
from src.logging_config import configure_logging
configure_logging(level="INFO")
logger = structlog.get_logger()

print("AI Automation Environment Verification")
print("=" * 50)

# 1. Config
api_key = os.getenv("GROQ_API_KEY")
if not api_key:
    print("FAIL: GROQ_API_KEY not found in .env")
    sys.exit(1)
logger.info("config_check", groq_key_present=True)
print("PASS: GROQ_API_KEY loaded")

# 2. Groq API
from groq import Groq
client = Groq()
response = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=[{"role": "user", "content": "Reply with exactly: AUTOMATION_OK"}],
    max_tokens=10,
    temperature=0,
)
reply = response.choices[0].message.content.strip()
logger.info("groq_check", response=reply)
print(f"PASS: Groq API response: '{reply}'")

# 3. Pydantic model validation
from pydantic import BaseModel, field_validator

class AgentAction(BaseModel):
    tool: str
    inputs: dict
    reasoning: str

    @field_validator("tool")
    @classmethod
    def tool_must_be_non_empty(cls, v: str) -> str:
        if not v.strip():
            raise ValueError("tool name cannot be empty")
        return v.strip()

action = AgentAction(
    tool="web_search",
    inputs={"query": "test"},
    reasoning="Verifying setup",
)
logger.info("pydantic_check", action=action.model_dump())
print(f"PASS: Pydantic model: tool='{action.tool}'")

# 4. Tenacity retry decorator
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=1, max=4))
def flaky_operation() -> str:
    return "SUCCESS"

result = flaky_operation()
print(f"PASS: Tenacity retry: '{result}'")

print("\n" + "=" * 50)
print("All components verified successfully.")
print("Ready to build AI automation systems.")
```

## Summary

- Install Python via **pyenv** and use **uv** to manage the automation stack: groq, pydantic, structlog, opentelemetry-sdk, httpx, and tenacity install cleanly together.
- Configure your **Groq API key** in `.env` and load it through **Pydantic Settings** for type-safe, validated configuration — this catches missing environment variables at startup, not mid-pipeline.
- Use **structlog** with `JSONRenderer` from the first line of your agent: every tool call, decision, and error becomes a structured log record you can search and filter in any observability platform.
- Instrument agent steps with **OpenTelemetry** spans to get a timeline of every operation — essential for debugging multi-step agent pipelines where a single tool call failure can cascade silently.
- Run **n8n** with Docker Compose as your visual automation layer; it handles HTTP webhooks, scheduling, and third-party integrations, letting your Python agents focus on intelligence rather than plumbing.
